# COLLEGEBUDDY PROJECT DOCUMENTATION

Project Name:
CollegeBuddy

Project Type:
Smart Campus Event Management System

Goal:
CollegeBuddy is a full-stack platform that allows students to discover campus events, hosts to create and manage events, and volunteers to scan QR-based tickets for attendance tracking. The system also includes AI-based event attendance prediction.

---

1. TECH STACK

---

Frontend:

* React (Vite)
* TailwindCSS
* React Router
* Axios (API communication)
* html5-qrcode (QR scanning)
* Lucide React (icons)

Backend:

* FastAPI (Python)
* SQLAlchemy ORM
* JWT Authentication
* Passlib (password hashing)
* Python-JOSE (JWT token handling)

Database:

* PostgreSQL

AI / ML:

* Scikit-learn
* Linear Regression Model
* Joblib (model storage)

Other Libraries:

* qrcode (QR generation)
* uuid (token generation)
* shutil (file handling)

4. USER ROLES

---

There are three user roles:

1. Student
2. Host
3. Volunteer

Student:

* Registers on platform
* Browses available events
* Books tickets
* Receives QR ticket

Host:

* Creates events
* Uploads posters
* Manages events
* Edits events
* Deletes events
* Views analytics

Volunteer:

* Uses scanner
* Scans QR codes
* Marks attendance

---

5. CURRENT FEATURES IMPLEMENTED

---

## Authentication System

* User registration
* User login
* JWT authentication
* Role-based login (student, host, volunteer)

## Event System

* Create event
* Edit event
* Delete event
* View all events
* Host-specific events

## Poster Upload

Hosts can upload event posters which are stored on the backend server.

## Event Registration

Students can:

* Browse events
* Book event tickets
* System generates unique QR code

## QR Ticket System

Each registration generates:

* Unique QR token
* QR code image
* Stored in database

## QR Scanner

Volunteers scan QR tickets using:
html5-qrcode

Scanner verifies:

* Ticket validity
* Marks attendance

## AI Attendance Prediction

Machine learning model predicts expected attendance based on:

Input:

* participant_limit
* event_fee
* total_registrations

Output:

* predicted attendance

Model type:
Linear Regression

6. API ENDPOINTS

---

Authentication

POST /register
Create new user account

POST /login
Login and receive JWT token

Events

POST /create-event
Create new event (host only)

GET /events
Fetch all events

GET /host/events
Fetch host's events

PUT /edit-event/{id}
Update event

DELETE /delete-event/{id}
Delete event

Event Registration

POST /register-event
Register student for event
Generate QR ticket

Attendance

POST /scan-ticket
Scan QR and mark attendance
